The Challenge

Why We're Making This Change

Firebase Storage served us well — but it's a general database tool, not a media platform.

💸
Rising Costs
Firebase charges increase as your member base grows, with no generous free tier for storage.
🐢
No CDN Delivery
Images are served from a single region, meaning slower loading for users on different connections.
🔧
Limited Media Features
Firebase lacks image-specific tools — no auto-compression, no resizing, no media management.

The Process

A Safe, Two-Phase Migration

Existing photos are migrated first — before the new code goes live. The app never goes offline.

01
Move Existing Photos
  • 1
    Read all records from Google Sheets
  • 2
    Download each photo from Firebase
  • 3
    Re-upload to Cloudinary, organised by category
  • 4
    Update each row with the new Cloudinary link
🟢  App stays live throughout
02
Switch New Uploads to Cloudinary
  • 1
    Update the app's upload function
  • 2
    Deploy updated app to Netlify
  • 3
    All new photos now go to Cloudinary
  • 4
    Firebase Storage safely retired
🔵  Seamless, instant cutover
